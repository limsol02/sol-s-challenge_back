### 1. 흥미로운 도입
광대한 도서관에 들어섰다고 생각해봅시다 이 도서관에는 수많은 책들이 있으며, 각 책에는 중요한 정보가 들어있습니다.
하지만 이 정보들을 찾고 싶다면, 책들이 어떻게 정리되어 있는지, 어디에 무엇이 있는지 알아야합니다. 
이것이 바로 데이터베이스 설계의 핵심입니다! 오라클 데이터베이스 설계는 마치 이 거대한 도서관의 
책들을 체계적으로 관리하는 방법을 배우는 것과 같습니다.

### 2. 학습목표
- 오라클 데이터베이스의 기본 구조 이해
- 효과적인 데이터베이스 설계 방법 학습
- 데이터 정규화 및 테이블 관계 설정
- SQL 쿼리 작성 및 최적화 기법 이해

### 3. 주효 핵심 키워드와 개념
- **ERD(Entity-Relationship Diagram)** : 데이터 모델을 시각화하는 도구
- **정규화(Normalization)** : 데이터 중복을 줄이고 무결성을 높이는 과정
- **테이블(Table)** : 데이터를 저장하는 기본 단위
- **기본키(Primary Key)** : 각 레코드를 유일하게 식별하는 키
- **외래키(Foreign Key)** : 다른 테이브르이 기본키를 참조하는 키
- **인덱스(Index)** : 데이터 검색 속도를 향상시키는 구조

### 4. 생각하고 토론해볼 내용
- 데이터베이스를 설계할 때 가장 중요한 요소는 무엇인가요?
- 정규화와 성능 사이에서 어떻게 균형을 맟출 수 있을까요?
- 대용량 데이터를 다룰 때 어떤 고려사항이 필요한가요?

3. ERD는 왜 중요한가요?

### 6. 실무적용 예시

- **병원 관리 시스템** : 환자, 의사, 예약 정보를 포함하는 데이터베이스 설계
